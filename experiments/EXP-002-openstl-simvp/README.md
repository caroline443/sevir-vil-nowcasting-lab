# EXP-002: pinned official OpenSTL SimVP compatibility gate

Status: completed — compatibility gate passed

## Question

Can the official OpenSTL SimVP architecture and SEVIR configuration complete one real-data mixed-precision optimization step in the existing Python 3.12 / PyTorch 2.8 / CUDA 12.8 environment, without downgrading the working GPU stack?

This is a compatibility and memory gate, not a full baseline training run and not an innovation experiment.

## Upstream code and protocol

- OpenSTL is pinned to commit `eecf8a3078f0a178dbc7b28723da20f94ce36985` (2025-10-22).
- The model uses OpenSTL's official SEVIR SimVP configuration: `IncepU`, `hid_S=64`, `hid_T=256`, `N_S=2`, `N_T=4`.
- OpenSTL's SimVP model returns the same number of frames it receives. Its official method wrapper handles 13→12 by retaining the first 12 predicted frames; this probe applies the same branch.
- The data protocol follows Earthformer's official SEVIR setup: 25-frame samples, starts `0/12/24`, 13 input frames, 12 targets, `pct_missing=0`, duplicated VIL IDs removed, training through 2019-01-01, validation through 2019-06-01, and later events for testing.
- EXP-001 used the same dates but assigned timestamps exactly on a boundary differently. The manifest builder is corrected in EXP-002, so the manifest must be rebuilt.

## 1. Create an isolated compatibility environment

Run from the repository root. This environment reuses the already working CUDA-enabled PyTorch installation and installs the pinned probe dependencies locally, without replacing Torch:

```bash
python -m venv --system-site-packages .venv-openstl
source .venv-openstl/bin/activate
python -m pip install --upgrade pip
python -m pip install --no-deps -r requirements-openstl.txt
python -m pip install -r requirements-data.txt
```

Do not run OpenSTL's unpinned full `requirements.txt` in the existing `zyx` environment.

## 2. Rebuild the official-protocol manifest

```bash
python scripts/build_sevir_manifest.py \
  --catalog /home/amon/zyx/dataset/sevir_data/CATALOG.csv \
  --output artifacts/local/sevir_official_manifest.csv \
  --train-end 2019-01-01 \
  --val-end 2019-06-01
```

The expected starts are the defaults `0,12,24`. Do not reuse `sevir_dev_manifest.csv` from EXP-001 because its boundary semantics were generated by the earlier code.

## 3. Run one real-data official-model step at 128×128

```bash
PYTHONPATH=src python scripts/smoke_openstl_simvp.py \
  --manifest artifacts/local/sevir_official_manifest.csv \
  --data-root /home/amon/zyx/dataset/sevir_data \
  --output artifacts/local/exp002_openstl_128.json \
  --resolution 128 \
  --batch-size 1
```

Return `artifacts/local/exp002_openstl_128.json`. Do not start full training yet.

The A4000 run completed successfully on 2026-07-15. The pinned upstream source hash matched, one real-data AMP optimization step took 0.94 seconds, and peak allocated GPU memory was approximately 1.06 GiB. See [`result.json`](result.json).

## Success condition

- raw official output is `[1, 13, 1, 128, 128]` and the protocol output is `[1, 12, 1, 128, 128]`;
- one AMP forward/backward/optimizer step succeeds with finite loss;
- peak allocated memory remains below 12 GiB;
- the report records OpenSTL, timm, SciPy, Torch and TorchVision versions.

## Stop condition

- If installation or import fails, return the complete error; do not downgrade PyTorch or CUDA.
- If the step exceeds 12 GiB or raises OOM, do not shrink the official architecture. The next gate will use activation checkpointing or spatial tiling at 384×384.
- A successful 128×128 step authorizes only a later 384×384 memory probe, not a multi-epoch training run.

## Outcome

Python 3.12.13, PyTorch 2.8.0+cu128, TorchVision 0.23.0+cu128 and the pinned OpenSTL SimVP source are compatible on the laboratory A4000. EXP-002 authorizes EXP-003's single-step native-resolution probe. The initial loss is recorded only as a finite-value check and has no forecasting significance.
